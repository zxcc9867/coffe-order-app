# 프론트엔드 화면 PRD (유저·관리자)

이 문서는 커피 주문 앱의 **주문하기(유저)** 화면과 **관리자** 화면에 대한 UI/UX 요구사항을 하나의 파일로 정의한다.

---

# Part I. 주문하기 화면 (유저)

## 1. 개요

### 1.1 목적
고객이 메뉴를 선택하고 옵션을 추가한 뒤 장바구니에 담고, 최종 주문을 완료하는 흐름을 담는다.

### 1.2 화면 위치
- 앱 내 2개 화면 중 하나: **주문하기** | 관리자
- 현재 화면: 주문하기 (고객 주문용)

---

## 2. 화면 구성

### 2.1 전체 레이아웃

| 구역 | 위치 | 설명 |
|------|------|------|
| 상단 내비게이션 | 최상단 | 브랜드명, 주문하기/관리자 탭 |
| 상품 목록 | 중앙 | 메뉴 카드 가로 배치 |
| 장바구니 | 하단 | 담긴 항목, 총 금액, 주문 버튼 |

---

## 3. 상단 내비게이션 (주문하기 화면)

### 3.1 요소

| 요소 | 위치 | 규격 |
|------|------|------|
| 브랜드명 | 좌측 상단 | 텍스트 "COZY" |
| 탭 | 우측 상단 | "주문하기", "관리자" |
| 선택 상태 | 주문하기 탭 | 현재 화면일 때 선택 표시 |

### 3.2 동작
- **주문하기** 탭: 현재 주문하기 화면 유지(또는 진입).
- **관리자** 탭: 관리자 화면으로 이동.

---

## 4. 상품 목록

### 4.1 레이아웃
- 카드형 상품이 **가로**로 나열.
- 한 행에 여러 카드(스크롤 또는 반응형 그리드).

### 4.2 상품 카드 구성

각 카드는 아래 요소를 포함한다.

| 영역 | 요소 | 비고 |
|------|------|------|
| 상단 | **이미지** | 상품 썸네일(플레이스홀더 가능) |
| 제목 | **상품명** | 예: 아메리카노(ICE), 아메리카노(HOT), 카페라떼 |
| 가격 | **기본 가격** | 원화 표기(예: 4,000원, 5,000원) |
| 설명 | **간단한 설명** | 1줄 요약(선택) |
| 옵션 | **체크박스** | 추가 옵션 다중 선택 |
| 액션 | **담기 버튼** | 클릭 시 장바구니에 추가 |

### 4.3 옵션 스펙

- **샷 추가**: 선택 시 +500원.
- **시럽 추가**: 선택 시 +0원.
- 옵션은 체크박스로 다중 선택 가능.
- 옵션 선택 후 "담기" 클릭 시, 선택된 옵션과 수량(1개)이 반영되어 장바구니에 추가된다.

### 4.4 예시 메뉴(와이어프레임 기준)

| 상품명 | 기본 가격 | 옵션 |
|--------|-----------|------|
| 아메리카노(ICE) | 4,000원 | 샷 추가 (+500원), 시럽 추가 (+0원) |
| 아메리카노(HOT) | 4,000원 | 샷 추가 (+500원), 시럽 추가 (+0원) |
| 카페라떼 | 5,000원 | 샷 추가 (+500원), 시럽 추가 (+0원) |

---

## 5. 장바구니

### 5.1 위치 및 형태
- 화면 **하단** 고정 또는 스크롤 시 노출.
- "장바구니" 제목이 있는 박스/패널.

### 5.2 장바구니 항목 표시

각 항목은 다음 정보를 표시한다.

| 항목 | 설명 | 예시 |
|------|------|------|
| 상품명 + 옵션 | 메뉴명과 적용된 옵션 텍스트 | 아메리카노(ICE) (샷 추가) |
| 수량 | X N 형식 | X 1, X 2 |
| 금액 | 해당 항목 소계(옵션 포함) | 4,500원, 8,000원 |

**금액 계산**
- 항목 금액 = (기본 가격 + 선택 옵션 추가 금액) × 수량
- 예: 아메리카노(ICE) + 샷 추가 1개 → 4,000 + 500 = 4,500원
- 예: 아메리카노(HOT) 2개(옵션 없음) → 4,000 × 2 = 8,000원

### 5.3 총 금액
- 라벨: "총 금액".
- 값: 장바구니 전체 합계를 **굵은 글씨**로 표시.
- 단위: 원(원화).
- 예: **12,500원**

### 5.4 주문 버튼
- 버튼 문구: "주문하기".
- 위치: 총 금액 아래.
- 동작: 클릭 시 주문 요청 전송(백엔드 연동).

---

## 6. 사용자 플로우 (주문하기)

1. **주문하기** 탭에서 메뉴 확인.
2. 원하는 메뉴 카드에서 옵션(샷/시럽 등) 선택.
3. **담기** 클릭 → 해당 메뉴(옵션 포함)가 장바구니에 1개 추가.
4. 필요 시 같은 메뉴를 다시 담아 수량 증가.
5. 하단 장바구니에서 항목과 **총 금액** 확인.
6. **주문하기** 버튼 클릭 → 주문 완료 처리.

---

## 7. 주문하기 화면 요구사항 요약

### 7.1 필수 요소
- 상단: COZY 브랜드, 주문하기/관리자 탭.
- 상품 카드: 이미지, 상품명, 가격, 설명, 옵션 체크박스, 담기 버튼.
- 장바구니: 항목별 상품명·옵션·수량·금액, 총 금액, 주문하기 버튼.

### 7.2 계산 규칙
- 옵션 추가 금액을 반영한 항목 단가 × 수량 = 항목 소계.
- 모든 항목 소계 합 = 총 금액.

### 7.3 추후 확장(선택)
- 장바구니 항목별 수량 변경/삭제.
- 옵션별 추가 금액을 백엔드/DB에서 관리.
- 주문하기 클릭 후 주문 완료/대기 상태 표시(관리자 화면과 연동).

# Part II. 관리자 화면

## 8. 개요

### 8.1 목적
관리자가 주문 현황을 확인·처리하고, 음료별 재고를 조절할 수 있도록 한다.

### 8.2 화면 위치
- 앱 내 2개 화면 중 하나: 주문하기 | **관리자**
- 현재 화면: 관리자 (운영·관리용)

---

## 9. 화면 구성

### 9.1 전체 레이아웃

| 구역 | 위치 | 설명 |
|------|------|------|
| 상단 내비게이션 | 최상단 | 브랜드명, 주문하기/관리자 탭 |
| 관리자 대시보드 | 상단 | 주문 요약 지표 |
| 재고 현황 | 중앙 | 음료별 재고 카드 및 수량 조절 |
| 주문 현황 | 하단 | 주문 목록 및 접수/처리 액션 |

---

## 10. 상단 내비게이션 (관리자 화면)

### 10.1 요소

| 요소 | 위치 | 규격 |
|------|------|------|
| 브랜드명 | 좌측 상단 | 텍스트 "COZY" |
| 탭 | 우측 상단 | "주문하기", "관리자" |
| 선택 상태 | 관리자 탭 | 현재 화면일 때 선택(활성) 표시 |

### 10.2 동작
- **주문하기** 탭: 주문하기 화면으로 이동.
- **관리자** 탭: 현재 관리자 화면 유지(또는 진입).

---

## 11. 관리자 대시보드

### 11.1 제목
- 표시 문구: **"관리자 대시보드"**

### 11.2 주문 요약 정보

한 줄 텍스트로 주문 상태별 건수를 표시한다.

| 항목 | 설명 | 예시 표기 |
|------|------|-----------|
| 총 주문 | 전체 주문 건수 | 총 주문 N |
| 주문 접수 | 접수 완료된 주문 수 | 주문 접수 N |
| 제조 중 | 제조 진행 중인 주문 수 | 제조 중 N |
| 제조 완료 | 완료된 주문 수 | 제조 완료 N |

**표시 형식 예시**
- "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"
- 구분자: " / " (공백 포함 슬래시)

### 11.3 동작 요구사항
- 주문 접수·제조 상태 변경 시 **실시간(또는 갱신 시점)** 으로 숫자가 반영되어야 함.
- 백엔드 주문 상태 API와 연동하여 데이터 표시.

---

## 12. 재고 현황

### 12.1 제목
- 표시 문구: **"재고 현황"**

### 12.2 레이아웃
- 음료별로 **카드** 형태로 나열.
- 카드 내: 재고 수량 표시 + 수량 조절 버튼(+, -).

### 12.3 카드별 구성

| 요소 | 설명 | 비고 |
|------|------|------|
| 음료명 | 메뉴 이름 | 예: 아메리카노(ICE), 아메리카노(HOT), 카페라떼 |
| 현재 재고 | 숫자 + 단위 | 예: "10개" |
| 감소 버튼 | "-" 버튼 | 클릭 시 해당 음료 재고 1 감소 |
| 증가 버튼 | "+" 버튼 | 클릭 시 해당 음료 재고 1 증가 |

### 12.4 예시(와이어프레임 기준)

| 음료 | 초기 재고 | 조절 |
|------|-----------|------|
| 아메리카노(ICE) | 10개 | + / - |
| 아메리카노(HOT) | 10개 | + / - |
| 카페라떼 | 10개 | + / - |

### 12.5 동작 요구사항
- **+** 클릭: 재고 1 증가 → 화면 및 백엔드 반영.
- **-** 클릭: 재고 1 감소 → 화면 및 백엔드 반영. (0 이하로는 내려가지 않도록 처리 권장)
- 재고는 메뉴(음료) 단위로 관리되며, 백엔드/DB와 동기화.

---

## 13. 주문 현황

### 13.1 제목
- 표시 문구: **"주문 현황"**

### 13.2 목록 형태
- 주문 건별로 **목록 아이템** 표시.
- 새로 들어온 주문·접수 대기 주문을 우선 노출.

### 13.3 주문 아이템 구성

각 주문 행에 다음 정보를 표시한다.

| 항목 | 설명 | 예시 |
|------|------|------|
| 주문 시간 | 주문 일시 | 7월 31일 13:00 |
| 주문 내용 | 음료명 + 수량 | 아메리카노(ICE) x 1 |
| 가격 | 주문 금액 | 4,000원 |
| 액션 버튼 | 주문 접수 | "주문 접수" 버튼 |

### 13.4 주문 접수 버튼
- 버튼 문구: **"주문 접수"**
- 위치: 해당 주문 행 옆(우측 권장).
- 동작: 클릭 시 해당 주문을 "주문 접수" 상태로 변경.
- 변경 후: 대시보드의 "주문 접수" 수가 증가하고, 필요 시 해당 주문은 "제조 중" 등 다음 단계로 이동하거나 목록에서 구분 표시.

### 13.5 상태 흐름(참고)
- (신규) → **주문 접수** → **제조 중** → **제조 완료**
- 대시보드 요약 숫자는 위 상태별 건수와 일치하도록 갱신.

---

## 14. 사용자 플로우 (관리자)

1. **관리자** 탭 선택 → 관리자 화면 진입.
2. **관리자 대시보드**에서 총 주문·접수·제조 중·완료 건수 확인.
3. **재고 현황**에서 음료별 재고 확인 후, 필요 시 **+** / **-** 로 수량 조절.
4. **주문 현황**에서 대기 주문 확인.
5. **주문 접수** 클릭 → 해당 주문 접수 처리 및 대시보드 숫자 갱신.
6. (추가 구현 시) 제조 중 → 제조 완료 등 상태 변경 버튼으로 진행.

---

## 15. 관리자 화면 요구사항 요약

### 15.1 필수 요소
- 상단: COZY 브랜드, 주문하기/관리자 탭(관리자 선택 상태).
- 대시보드: "관리자 대시보드" 제목, 주문 요약(총 주문 / 주문 접수 / 제조 중 / 제조 완료).
- 재고 현황: "재고 현황" 제목, 음료별 카드(재고 수량, + / - 버튼).
- 주문 현황: "주문 현황" 제목, 주문 목록(시간, 내용, 가격, 주문 접수 버튼).

### 15.2 데이터 일관성
- 대시보드 숫자와 주문 목록·상태는 백엔드와 동기화.
- 재고 변경은 즉시 서버에 반영하고, 동시 수정 시 정합성 유지 방안 고려.

### 15.3 추후 확장(선택)
- 주문 상태별 필터(접수 대기만 보기, 제조 중만 보기 등).
- "제조 중" → "제조 완료" 버튼을 주문 행에 추가.
- 재고 부족 시 주문하기 화면에서 해당 메뉴 품절 표시 또는 주문 제한.
- 재고 변경 이력 로그.

# Part III. 공통 사항

## 16. 공통 내비게이션

- 두 화면 모두 **상단에 동일한 내비게이션** 사용: 좌측 "COZY", 우측 "주문하기" | "관리자" 탭.
- 현재 화면에 해당하는 탭만 선택(활성) 상태로 표시.

## 17. 화면 간 연동

- 고객이 **주문하기** 화면에서 "주문하기" 버튼으로 제출한 주문이 **관리자** 화면의 "주문 현황"에 노출됨.
- 주문 접수·제조 완료 등 상태 변경은 (추후) 고객 대기 화면이나 알림과 연동 가능.

## 18. 공통 UI/UX

- **반응형**: 상품 카드·재고 카드·주문 목록은 화면 크기에 따라 배치 조정.
- **접근성**: 버튼, 탭, 체크박스는 키보드/포커스로 조작 가능하도록 구현 권장.
- **일관성**: 금액은 천 단위 구분(쉼표) 사용(예: 12,500원). 날짜·시간 형식은 앱 전역에서 통일.
